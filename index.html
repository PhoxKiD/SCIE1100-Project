Welcome to Kryptos

The purpose of Kryptos is to teach anyone the basics of Cryptology, along with some of the early applications of Cryptography, such as the Caesar cipher and the Vingenere cipher.

Encryption is a modern application of cryptology. Encrypting something means to alter it in another way using a key, so it becomes unreadable. It can be decrypted using the same key. Today, encryption is used to keep Internet connections secure and and files safe.

Caesar Cipher

The Caesar Cipher is one of the simplest forms of encryption. The method works by shifting the alphabet to the right by a certain number to cipher a message. For example, a number of 2 will turn every "A" in the original message to "C" in the encoded message.

Instructions: Use the draggable bar to select a number to shift the alphabet by. Input a message into the box and click Cipher to see the shifted message below the box. You can also input a shifted message and click Decipher to see the original message.

Vingenère

The Vingenère Cipher is a more complex form of the Ceaser Cipher. The cipher uses a codeword to turn the input into an encoded output. The output can be deciphered using the same codeword. Every letter of the input is matched with a letter from the codeword, and by using the large table below, a new letter can be created for the encoded message. The same process is done for the next letter in the input and the next letter in the codeword. If the codeword runs out of characters, then it goes back to the beginning of the codeword and continues. The process goes on until the entire input message has been encoded and outputted.

Encryption Instructions: Type the codeword used to cipher into the box on the left. Then type the message you want to encode into the box on the right and press enter. You will see the encoded output below the boxes.

I would never die for my beliefs because I might be wrong

Table Instructions: The letters on the rows and columns represent the characters of the input message and characters of the codeword. You can figure out the encoded message without the javascript algorithm. Find the column representing the 1st letter of the input message and the row representing the 1st letter of the codeword and find where they intersect. The letter they intersect at will be the 1st character of the encoded output message. Next, use the 2nd letter of your input and the 2nd character of your codeword and find the intersection on the table, which will be the 2nd letter of the output. You can do this for all letters of the input message. Just remember to go back to the 1st letter of the codeword should you run out of letters in the codeword to use. This can be done backwards too (deciphering or decoding the message), but it will be a little harder. Find the row representing the 1st letter of the codeword and run along that row until you come across the 1st letter of the encoded message. The column highlighted will be the 1st letter of the original message. Then, use the 2nd letter of the encoded output message and the second letter of the codeword, and the highlighted column will be the 2nd letter of the original message. You can do this for the whole encoded message, while going back to the 1st character of the codeword should you run out of letters to use in the codeword.
